# FILE SCANNER v1.0 - Tab 1: 키워드 문서 검색기

> 문서 상태: 설계 명세 (구현 목표 기준)

프로그램 개요

지정한 드라이브/폴더 내 문서 파일을 스캔하여 사용자 정의 키워드가 포함된 파일을 찾아 위치와 해당 문장을 보고합니다.

프로젝트 구조 (구현 완료 목표)

D:\VS\FILE_SCANNER\
├── scanner_engine.py      ← 파일 탐색, 텍스트 추출, 키워드 검색
├── scanner_gui.py         ← CustomTkinter GUI (탭 구조)
├── keywords.json          ← 키워드 저장 (자동 생성)
├── venv/
└── .gitignore

scanner_engine.py 설계

파일 탐색 함수 scan_drive(paths, extensions): 지정된 경로 리스트를 재귀 탐색하여 대상 확장자 파일 목록 반환. Windows, Program Files, $Recycle.Bin, venv, node_modules, .git 등 시스템/개발 폴더는 자동 제외.

텍스트 추출: 기존 공사명칭 검토기의 추출 함수를 재활용. PDF(PyMuPDF), Excel(.xlsx: openpyxl, .xls: xlrd), Word(.docx: python-docx, .doc: olefile/WordDocument 스트림), HWP(olefile), HWPX(zipfile+xml), CSV(내장 csv) 지원.

키워드 검색 함수 search_keywords(text_items, keywords): 추출된 텍스트에서 키워드 포함 여부를 확인하고, 키워드별로 파일명, 위치(페이지/행/셀), 해당 문장을 반환.

키워드 관리 함수: load_keywords(path="keywords.json") → 파일에서 키워드 리스트 로드. save_keywords(keywords, path="keywords.json") → 키워드 리스트를 JSON으로 저장.

scanner_gui.py 설계

CustomTkinter 기반, 탭 구조(CTkTabview)로 향후 2번·3번 탭 추가 가능.

주요 기능 상세

드라이브/폴더 선택: 시스템 드라이브를 자동 감지하여 체크박스로 표시. [폴더 추가] 버튼으로 특정 폴더를 추가 선택 가능. 중복 선택 허용 (C:\ 전체 + C:\Users\특정폴더 → 중복 파일은 자동 제거).

키워드 관리: 입력란에 키워드 입력 후 [추가] 클릭 → 리스트에 추가. 리스트에서 선택 후 [삭제] 클릭 → 제거. [저장] 클릭 → keywords.json에 저장. 프로그램 시작 시 keywords.json이 있으면 자동 로드.

검색 실행: [검색 시작] 클릭 → 백그라운드 스레드에서 실행. 진행률 바와 파일 카운트 실시간 표시. 검색 중 [검색 중지] 버튼으로 변경, 클릭 시 중단.

결과 표시: 키워드별로 그룹핑하여 로그 영역에 출력. 형식 예시:

[보안] C:\Users\...\보고서.pdf (P3 L12)
  "...본 문서는 보안 등급 2급으로 분류되며..."

[기밀] D:\공유폴더\회의록.hwp (HWP L45)
  "...기밀 사항이므로 외부 유출 금지..."
리포트 저장: Excel 파일로 저장. 컬럼: 키워드, 파일경로, 위치, 해당문장, 검색일시.

대상 확장자

.pdf, .xlsx, .xls, .docx, .doc, .hwp, .hwpx, .csv, .txt

.txt는 신규 추가 (단순 텍스트 읽기, open(f, encoding=...) with fallback).

성능 고려

파일 탐색과 텍스트 추출을 분리: 먼저 전체 파일 목록을 수집(빠름) → 진행률 계산 가능 → 순차 추출+검색. 대용량 드라이브(수만 파일) 대비 os.scandir() 사용 (os.walk 대비 빠름). 텍스트 추출 실패 시 건너뛰고 에러 로그만 기록. 검색은 단순 keyword in text (대소문자 무시) — 정규식보다 빠름.

변경 금지 (향후 탭 추가 시)

탭 구조 (CTkTabview), 공통 UI 프레임, scanner_engine.py의 추출 함수.
